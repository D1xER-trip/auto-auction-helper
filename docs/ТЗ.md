# Техническое задание на разработку сайта помощи в покупке автомобилей с аукционов и доставке в РФ

## 1. Общая информация

**Название проекта:** (CarCollection)  
**Цель проекта:** Создание веб-платформы, которая позволяет клиентам подбирать, выкупать и доставлять автомобили с зарубежных аукционов (США, Япония, Корея) в Российскую Федерацию «под ключ».

**Задачи:**
- Предоставить удобный интерфейс для расчёта полной стоимости авто с учётом всех расходов ( доставка, пошлины, комиссии).
- Интегрироваться с данными аукционов для отображения актуальных лотов.
- Автоматизировать процессы подачи заявок, отслеживания статуса, документооборота.
- Привлекать клиентов через информационный контент (блог, статьи, калькуляторы).

## 2. Целевая аудитория

- **Частные лица:** жители РФ, желающие приобрести авто из-за рубежа для личного пользования. Возраст 25–55 лет, доход выше среднего, наличие опыта онлайн-покупок.
- **Автоперекупщики и малые автосалоны:** предприниматели, нуждающиеся в регулярных поставках автомобилей.
- **Опытные покупатели:** те, кто уже покупал авто самостоятельно, но нуждается в услугах (проверка, логистика).


## 3. Функциональные требования

### 3.1. Публичная часть (для всех посетителей)

- **Главная страница:**
  - Краткое описание услуг.
  - Блок с преимуществами.
  - Форма быстрого расчёта стоимости (калькулятор).
  - Последние статьи из блога / отзывы.
- **Калькулятор стоимости «под ключ»:**
  - Выбор страны аукциона (США, Япония, Корея).
  - Выбор марки, модели, года, состояния (пробег, повреждения).
  - Расчёт ориентировочной финальной цены (ставка + аукционные сборы + доставка + таможенные платежи + комиссия сервиса).
  - Возможность сохранить расчёт и получить на email / в личном кабинете.
- **Каталог лотов (интеграция с аукционами):**
  - Отображение актуальных лотов с фильтрацией (марка, модель, год, цена, страна).
  - Детальная страница лота: фото, описание, история,текущая ставка, время до окончания торгов.
  - Кнопка «Заказать проверку» / «Участвовать в торгах».
- **Блог / База знаний:**
  - Статьи, гайды, видеоинструкции.
  - Разбор нюансов покупки, таможни, логистики.
  - Возможность подписки на новости.
- **Страницы услуг:**
  - Подбор авто.
  - Участие в торгах.
  - Проверка истории.
  - Логистика и доставка.
  - Таможенное оформление.
  - Страхование.
- **Контакты и обратная связь:**
  - Форма связи, телефон, мессенджеры.
- **Отзывы:**
  - Реальные примеры покупок с фото и видео.
  - Форма для оставления отзыва.

### 3.2. Личный кабинет клиента (авторизованная зона)

- Регистрация / авторизация (по email, номеру телефона, через соцсети).
- Профиль пользователя: контактные данные, предпочтения.
- Мои заявки (история обращений, статусы).
- Сохранённые расчёты и избранные лоты.
- Уведомления (о новых ставках, изменении статуса доставки).
- Документооборот: загрузка документов, подписание договоров (возможно, через интеграцию с ЭДО).
- Финансовый раздел: счета, оплаты, история транзакций.

### 3.3. Административная панель (для менеджеров)

- Управление пользователями (просмотр, блокировка).
- Управление заявками (назначение ответственного, смена статуса, комментарии).
- Наполнение контентом (статьи, лоты вручную, отзывы).
- Настройка калькулятора (тарифы, пошлины, курсы валют, комиссии).
- Интеграция с аукционами (обновление данных).
- Просмотр статистики и отчётов.

### 3.4. Интеграции

- **С аукционами:** Copart, IAAI, USS, etc.
- **С сервисами проверки истории:** Carfax, Autocheck.
- **С платёжными системами:** приём оплат (возможно, через ЮKassa, Robokassa или др.).
- **С CRM-системой** (например, amoCRM или самописная).
- **С мессенджерами:** Telegram-бот для уведомлений.
- **С логистическими сервисами:** отслеживание грузов (трекинг).

## 4. Нефункциональные требования

- **Производительность:** время загрузки страниц не более 2–3 секунд, оптимизация изображений, кэширование.
- **Безопасность:** HTTPS, защита от SQL-инъекций, XSS, CSRF. Хранение паролей в хешированном виде. Регулярное резервное копирование.
- **Адаптивность:** сайт должен корректно отображаться на всех типах устройств (десктоп, планшет, смартфон).
- **Кроссбраузерность:** поддержка последних версий Chrome, Firefox, Safari, Edge, а также Яндекс.Браузера.
- **Юзабилити:** интуитивно понятный интерфейс, наличие подсказок, минимизация количества шагов для оформления заявки.
- **Масштабируемость:** архитектура должна позволять добавлять новые функции без переписывания ядра.

## 5. Требования к дизайну

- Стиль: современный, технологичный, вызывающий доверие .
- Акцент на фотографии автомобилей.
- Удобная типографика, понятная иерархия.
- Наличие элементов геймификации (прогресс-бары, статусы) для удержания внимания.
- Логотип и фирменный стиль .

## 6. Технический стек (предварительный)

- **Бэкенд:** Python (Django )
- **Фронтенд:** HTML5, CSS3, JavaScript (возможно, с использованием Vue.js или React для интерактивных частей).
- **База данных:** MySQL / PostgreSQL.
- **Сервер:** Linux (Ubuntu), Apache.
- **Версионный контроль:** Git (GitHub / GitLab).
- **Инфраструктура:** Docker для разработки, CI/CD для автоматизации деплоя.

## 7. Этапы разработки

1. **Аналитика и прототипирование:** составление карты пользовательских путей, создание прототипов (wireframes).
2. **Дизайн:** разработка макетов ключевых страниц.
3. **Вёрстка и фронтенд:** адаптивная вёрстка, подключение интерактива.
4. **Разработка бэкенда:** создание архитектуры, базы данных, интеграции.
5. **Интеграция с внешними сервисами:** аукционы, платёжные системы и т.д.
6. **Тестирование:** функциональное, нагрузочное, юзабилити.
7. **Запуск (MVP):** минимально жизнеспособный продукт (калькулятор, каталог лотов, личный кабинет).
8. **Сбор обратной связи и доработка.**
9. **Полноценный запуск и маркетинговое продвижение.**

## 8. Ожидаемые результаты

- Рабочий сайт, доступный по адресу.
- Возможность для клиентов самостоятельно рассчитывать стоимость, просматривать лоты, оставлять заявки.
- Инструменты для менеджеров по обработке заявок.
- Повышение прозрачности услуг и доверия к компании.
